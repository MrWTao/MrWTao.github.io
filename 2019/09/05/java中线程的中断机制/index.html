java中线程的中断机制

发表于 2019-09-05 | 分类于 java并发编程

线程interrupt

线程的interrupt是一个非常重要的API，与interrupt相关的API有如下几个：

interrupt()方法

  1. 线程在处于阻塞状态的时候调用线程的interrupt()方法会打断阻塞状态，使线程进入阻塞状态的方法成为可中断方法，这里需要注意一点就是打断一个线程并不意味这线程的生命周期结束，仅仅是打断了当前线程的阻塞状态而已；
  2. 线程在阻塞状态下被打断会抛出一个InterruptedException异常，这个异常就像信号一样通知这个线程被打断了。

我们可以查看interrupt()方法的源码，发现该方法中断线程之后会根据情况判断是否重新设置线程的中断状态为false。

isInterrupted()方法

isInterrupted()方法是Thread类中的一个成员方法，Thread类中有一个flag标识用于标识当前线程是否处于中断状态。isInterrupted()的作用是判断当前线程是否被打断，通过源码我们发现该方法仅仅是对Thread类中的interrupt标识的一个判断，并不会做其他的任何操作。

所以我们验证了线程在可中断状态和其他状态被打断的不同，在可中断状态被打断会重设线程的中断标识，这样是可以理解的，因为我们的线程出于阻塞状态被打断后，应该不去影响线程后面的代码的执行，所以会clear打断状态；而在正常不会抛出InterruptedException的状态下被打断仅仅是将打断状态置为true而已。
